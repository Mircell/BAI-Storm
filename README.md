# BAI-Storm - طوفان فکری چند ایجنت

یک ابزار تحت وب برای برگزاری جلسات طوفان فکری با استفاده از چندین عامل هوش مصنوعی (Agent) که به صورت تعاملی ایده‌های خود را ارائه می‌دهند. این پروژه از API‌های LLM (مانند OpenAI یا پروکسی‌های محلی مانند Ollama) برای تولید ایده‌های خلاقانه استفاده می‌کند و از دو روش اصلی پشتیبانی می‌کند: **طوفان فکری استاندارد** و **تکنیک شش کلاه تفکر**.

## ویژگی‌ها

- **چندین ایجنت هم‌زمان**: تعداد دلخواه ایجنت (۱ تا ۲۰) با شخصیت‌های قابل سفارشی‌سازی.
- **شخصیت‌سازی ایجنت‌ها**: برای هر ایجنت می‌توانید یک شخصیت و نقش تعریف کنید تا ایده‌های متنوع‌تری دریافت کنید.
- **دو حالت جلسه**:
  - **طوفان فکری استاندارد**: ایجنت‌ها به نوبت ایده‌های جدید ارائه می‌دهند.
  - **تکنیک شش کلاه تفکر (Six Thinking Hats)**: شش ایجنت با شش دیدگاه متفاوت (سفید، قرمز، سیاه، زرد، سبز، آبی) مسئله را از زوایای مختلف بررسی می‌کنند.
- **ذخیره‌سازی جلسات**: جلسات به صورت موقت در حافظه سرور نگهداری می‌شوند.
- **خروجی HTML**: امکان دریافت خلاصه جلسه به صورت یک صفحه HTML برای ذخیره یا چاپ.
- **رابط کاربری زیبا و واکنش‌گرا** (با استفاده از Bootstrap و آیکون‌های Bootstrap Icons).
- **تنظیمات انعطاف‌پذیر API**: امکان اتصال به هر API سازگار با OpenAI (مانند OpenAI رسمی، پروکسی‌های محلی، Ollama، و غیره).

## پیش‌نیازها

- Python 3.8 یا بالاتر
- pip (مدیریت بسته‌های Python)
- یک سرویس‌دهنده LLM (مانند OpenAI، پروکسی محلی، یا Ollama) با API سازگار.

## نصب و راه‌اندازی

۱. مخزن را کلون کنید:
   ```bash
   git clone https://github.com/Mircell/BAI-Storm.git
   cd BAI-Storm
   ```

۲. بسته‌های مورد نیاز را نصب کنید:
   ```bash
   pip install -r requirements.txt
   ```
   فایل `requirements.txt` شامل:
   - Flask 2.3.3
   - OpenAI 1.12.0 (اختیاری)
   - Requests 2.31.0

۳. سرور را اجرا کنید:
   ```bash
   python app.py
   ```

۴. برنامه در آدرس `http://localhost:5000` در دسترس خواهد بود.

## پیکربندی

پس از اجرا، در صفحه اصلی می‌توانید تنظیمات زیر را پیکربندی کنید:

- **Base URL**: آدرس API مدل (پیش‌فرض: `http://127.0.0.1:8000/v1` برای پروکسی محلی)
- **API Key**: کلید API (برای پروکسی محلی می‌توانید هر مقداری مانند `sk-local` وارد کنید)
- **Model**: نام مدل (مانند `thinking_not_search`، `llama2`، `gpt-3.5-turbo` و غیره)
- **تعداد ایجنت‌ها**: تعداد عوامل شرکت‌کننده در جلسه
- **مسئله/مشکل**: موضوع یا مشکل مورد نظر برای طوفان فکری
- **شخصیت‌ها**: برای هر ایجنت می‌توانید شخصیت و دستورالعمل خاصی تعریف کنید.

## استفاده

۱. در صفحه اصلی، تنظیمات مورد نظر را وارد کنید.
۲. دکمه **شروع طوفان فکری** را بزنید.
۳. جلسه آغاز می‌شود و ایجنت‌ها به نوبت ایده‌های خود را نمایش می‌دهند.
۴. در هر لحظه می‌توانید با دکمه **توقف** جلسه را متوقف کنید.
۵. پس از توقف یا تکمیل جلسه، دکمه **دریافت خروجی HTML** ظاهر می‌شود که با کلیک روی آن، خلاصه جلسه به صورت یک صفحه HTML در پنجره جدید باز می‌شود.

## تکنیک شش کلاه تفکر

با کلیک روی دکمه **تکنیک شش کلاه تفکر** در بالای صفحه، به حالت ویژه شش کلاه تفکر می‌روید. در این حالت، شش ایجنت با کلاه‌های زیر به ترتیب نظر خود را ارائه می‌دهند:

- **کلاه سفید** (اطلاعات و داده‌ها)
- **کلاه قرمز** (احساسات و شهود)
- **کلاه سیاه** (نقاط ضعف و ریسک‌ها)
- **کلاه زرد** (مزایا و فرصت‌ها)
- **کلاه سبز** (خلاقیت و ایده‌های نو)
- **کلاه آبی** (مدیریت و جمع‌بندی)

این روش به شما کمک می‌کند تا مسئله را از زوایای مختلف بررسی کنید.

```

│   ├── export.html         # قالب خروجی HTML برای حالت استاندارد

└── README.md               # این فایل
```

## نکات

- برنامه از یک ذخیره‌سازی موقتی در حافظه برای جلسات استفاده می‌کند (برای دمو).
- برای استفاده در تولید، بهتر است از یک دیتابیس پایدار استفاده کنید.
- تنظیمات API به صورت امن از طریق فرم ارسال می‌شوند و در سرور ذخیره نمی‌شوند.
- اگر از پروکسی محلی مانند Ollama استفاده می‌کنید، مطمئن شوید که سرور در حال اجرا است و آدرس Base URL صحیح است.
