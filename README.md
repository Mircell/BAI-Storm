# BAI-Storm - ابزار خلاقیت چند ایجنت

یک ابزار تحت وب برای برگزاری جلسات طوفان فکری و تحلیل خلاقانه با استفاده از چندین عامل هوش مصنوعی (Agent). این پروژه از API‌های LLM (مانند OpenAI یا پروکسی‌های محلی مانند Ollama) برای تولید ایده‌های خلاقانه استفاده می‌کند و از دو روش اصلی پشتیبانی می‌کند: **طوفان فکری استاندارد** و **تکنیک شش کلاه تفکر**.

## ویژگی‌ها

- **چندین ایجنت هم‌زمان**: تعداد دلخواه ایجنت (۱ تا ۲۰) با شخصیت‌های قابل سفارشی‌سازی.
- **نام‌گذاری دلخواه ایجنت‌ها**: به هر ایجنت می‌توانید یک نام دلخواه بدهید (اختیاری) تا تشخیص آن آسان‌تر باشد.
- **شخصیت‌سازی ایجنت‌ها**: برای هر ایجنت می‌توانید یک شخصیت و نقش تعریف کنید تا ایده‌های متنوع‌تری دریافت کنید.
- **دو حالت جلسه**:
  - **طوفان فکری استاندارد**: ایجنت‌ها به نوبت ایده‌های جدید ارائه می‌دهند.
  - **تکنیک شش کلاه تفکر (Six Thinking Hats)**: شش ایجنت با شش دیدگاه متفاوت (سفید، قرمز، سیاه، زرد، سبز، آبی) مسئله را از زوایای مختلف بررسی می‌کنند.
- **ذخیره‌سازی جلسات**: جلسات به صورت موقت در حافظه سرور نگهداری می‌شوند.
- **خروجی HTML پیشرفته**: امکان دریافت خلاصه جلسه به صورت یک صفحه HTML با قابلیت‌های:
  - نمایش **شماره دور** (نوبت) برای هر ایده.
  - **فیلتر بر اساس ایجنت یا کلاه** برای مشاهده فقط نظرات یک گوینده خاص.
  - نمایش رنگ و ایموجی کلاه‌ها در حالت شش کلاه.
- **رابط کاربری زیبا و واکنش‌گرا** با استفاده از Bootstrap و آیکون‌های Bootstrap Icons.
- **تنظیمات انعطاف‌پذیر API**: امکان اتصال به هر API سازگار با OpenAI (مانند OpenAI رسمی، پروکسی‌های محلی، Ollama، و غیره).
- **صفحه اصلی مرکزی** با لیست ابزارهای خلاقیت و تنظیمات یکپارچه API که در مرورگر ذخیره می‌شود.

## پیش‌نیازها

- Python 3.8 یا بالاتر
- pip (مدیریت بسته‌های Python)
- یک سرویس‌دهنده LLM (مانند OpenAI، پروکسی محلی، یا Ollama) با API سازگار.

## نصب و راه‌اندازی

۱. مخزن را کلون کنید:
   ```bash
   git clone https://github.com/Mircell/BAI-Storm.git
   cd BAI-Storm
   ```

۲. بسته‌های مورد نیاز را نصب کنید:
   ```bash
   pip install -r requirements.txt
   ```
   فایل `requirements.txt` شامل:
   - Flask 2.3.3
   - OpenAI 1.12.0 (اختیاری)
   - Requests 2.31.0

۳. سرور را اجرا کنید:
   ```bash
   python app.py
   ```

۴. برنامه در آدرس `http://localhost:5000` در دسترس خواهد بود.

## پیکربندی و استفاده

### صفحه اصلی
پس از اجرا، صفحه اصلی شامل دو بخش است:
- **تنظیمات AI API Provider**: مقادیر `Base URL`، `API Key` و `Model` را وارد کنید. این مقادیر به صورت خودکار در مرورگر ذخیره می‌شوند.
- **لیست ابزارهای خلاقیت**: دو کارت برای «طوفان فکری چند ایجنت» و «تکنیک شش کلاه تفکر» نمایش داده می‌شود. با کلیک روی هر کارت به صفحه آن ابزار هدایت می‌شوید.

### طوفان فکری استاندارد
- در صفحه `/brainstorm`، تعداد ایجنت‌ها، مسئله، و در صورت تمایل نام و شخصیت هر ایجنت را تنظیم کنید.
- با کلیک روی **شروع طوفان فکری**، جلسه آغاز می‌شود و ایجنت‌ها به نوبت ایده‌های خود را نمایش می‌دهند.
- در هر لحظه می‌توانید با دکمه **توقف** جلسه را متوقف کنید.
- پس از توقف یا تکمیل جلسه، دکمه **دریافت خروجی HTML** ظاهر می‌شود که خلاصه جلسه را با شماره دور و امکان فیلتر بر اساس ایجنت نمایش می‌دهد.

### تکنیک شش کلاه تفکر
- در صفحه `/six-hats`، ایده یا مسئله مورد نظر را وارد کنید.
- با کلیک روی **شروع تحلیل با شش کلاه**، شش کلاه به ترتیب نظر خود را ارائه می‌دهند.
- مانند حالت استاندارد، می‌توانید جلسه را متوقف کرده و خروجی HTML دریافت کنید. در خروجی، هر کلاه با رنگ و ایموجی مشخص شده و امکان فیلتر بر اساس کلاه وجود دارد.

```
BAI-Storm/
├── app.py                          # نقطه ورود برنامه Flask
├── config.py                       # تنظیمات (کلید مخفی، دیباگ، هاست، پورت، لاگ)
├── requirements.txt                # وابستگی‌های Python
├── models/                         # مدل‌های داده
│   └── session.py                  # کلاس‌های BrainstormSession و SixHatsSession
├── routes/                         # تعریف مسیرها (Routes)
│   └── main.py                     # کلیه مسیرها با استفاده از Blueprint
├── services/                       # سرویس‌های کمکی
│   └── session_manager.py          # مدیریت جلسات در حافظه (ایجاد، دریافت، توقف)
├── templates/                      # قالب‌های HTML
│   ├── main.html                   # صفحه اصلی (تنظیمات API + لیست ابزارها)
│   ├── brainstorm.html             # صفحه طوفان فکری استاندارد
│   ├── six_hats.html               # صفحه تکنیک شش کلاه تفکر
│   ├── export.html                 # قالب خروجی HTML برای حالت استاندارد
│   └── export_six_hats.html        # قالب خروجی HTML برای حالت شش کلاه
├── app.log                         # فایل لاگ (توسط Flask ایجاد می‌شود)
├── README.md                       # این فایل
├── .gitignore                      # فایل‌های نادیده‌گرفته‌شده توسط Git
└── memory.md                       # مستندات فنی برای توسعه‌دهندگان
```

## نکات

- برنامه از یک ذخیره‌سازی موقتی در حافظه برای جلسات استفاده می‌کند (برای دمو).
- برای استفاده در تولید، بهتر است از یک دیتابیس پایدار استفاده کنید.
- تنظیمات API به صورت امن از طریق فرم ارسال می‌شوند و در سرور ذخیره نمی‌شوند.
- اگر از پروکسی محلی مانند Ollama استفاده می‌کنید، مطمئن شوید که سرور در حال اجرا است و آدرس Base URL صحیح است.
